# Lab 5: Feature Engineering (Classification)

This repository contains my work for **Machine Learning Lab 5**, focusing on **Feature Engineering for a Classification Task**.

##  Lab Objective

The goal of this lab is to apply feature engineering techniques to improve model performance in a classification problem.

We aim to predict **Order Status** (Delivered, Cancelled, In Transit) using a Talabat-style orders dataset.

##  Lab Focus

In this lab:

- The dataset is already clean (no missing values or duplicates)
- The main focus is on **feature engineering**
- Understanding how features affect model performance

##  Objective

- Build a baseline classification model
- Predict `Order_Status`
- Analyze how feature engineering impacts:
  - Model accuracy
  - Feature importance

##  Tools and Libraries Used

- Python
- Google Colab
- Pandas
- NumPy
- Matplotlib
- Seaborn
- Scikit-learn

##  Files Included

- `Feature Engineering.ipynb`
- Dataset file (orders dataset)
- Model outputs and evaluation results

## Lab Tasks

In this lab, I performed:

- Importing required libraries
- Preparing the dataset
- Feature encoding (e.g., One-Hot Encoding)
- Splitting data into training and testing sets
- Building a classification model (Random Forest)
- Evaluating the model (accuracy, confusion matrix, classification report)
- Analyzing feature importance

## Conclusion

This lab demonstrated how feature engineering plays a critical role in improving machine learning models. Even with clean data, selecting and transforming the right features can significantly impact model performance.
